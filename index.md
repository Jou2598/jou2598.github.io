## 📋 Understanding the scoring system

The tool uses a scoring system from 0 to 100 to evaluate your project. A lower score indicates better design quality.

* AI Slop: This score measures how much your code relies on generic AI patterns. Aim for a score of 20 or below.
* Copy Slop: This score checks your landing page text. It flags repetitive or robotic phrasing. Aim for a score of 20 or below.
* Taste: This score evaluates your overall layout and design choices against industry standards. Aim for a score of 20 or below.

When all three categories stay at or below 20, your project meets the recommended quality standards.

## 📖 Managing project context

BeUniq uses two main files to track your project direction:

1. PRODUCT.md: This file tracks your product goals and target audience. Keeping this file updated helps the tool understand why you made certain design choices.
2. DESIGN.md: This file stores your style guidelines. Write down your preferred colors, spacing, and typography here. 

The tool references these files to ensure your project direction stays clear. It prevents the loss of your design intent after you finish a conversation with other AI assistants.

## 🔍 Frequently asked questions

Does this tool change my code automatically?
No. BeUniq is a diagnostic tool. It identifies issues and suggests improvements. You remain in full control of your actual files.

What if my project uses a framework not listed?
The tool focuses on standard web files. It works well with most modern web projects that use CSS or class-based styling.

Can I save my reports?
Yes. The application creates a text report folder within your project directory after every scan. You can open these files in any text editor to review the findings later.

What should I do if a rule flags my custom code?
Sometimes custom code triggers a flag. If you feel the code follows your project style, you can ignore that specific suggestion in the settings menu.
